User Input

Textboxes, radio buttons, check boxes, and dropdown lists.

<form class="" action="index.html" method="post">
<input type="__text__" name="" value="" required>
<button type="button" name="button" > button name </button >
</form>

Accessibility: Set a for attribute on the label element, with a value that matches the input element's ID attribute. Assistive tech uses this to create a connection between the label and its related input field.

Type can be checkboxes or radio buttons. But for checkboxes, but the label before the input tag. You may want to give everything classes or IDs.

<input type="__radio / checkbox__" name="yes-no" value="yes" checked>
<label for="__yes__""> yes </label>
<input type="__radio / checkbox__" name="yes-no" value="no">
<label for="__no__""> no </label>

When a form is submitted, the data is then sent to the server and includes for (for) the options selected. input types radio and checkbox report their values from the value attribute. The data recieved would be yes-no = yes

If the value attribute is ommitted, the submitted form uses the default value. Which is "on". That may not be ideal for your purposes. yes-no = yes is more intuitive that yes-no = on. Override this with the checked attribute. The required attribute makes input manditory.
