    • User Input

    Textboxes, radio buttons, check boxes, and dropdown lists.

    <form class="" action="index.html" method="post">
    <input type="__text__" name="" value="" required>
    <button type="button" name="button" > button name </button >
    </form>

    Accessibility: Set a for attribute on the label element, with a value that matches the input element's ID attribute. Assistive tech uses this to create a connection between the label and its related input field.

    Type can be checkboxes or radio buttons. But for checkboxes, but the label before the input tag. You may want to give everything classes or IDs.

    <input type="__radio / checkbox__" name="yes-no" value="yes" checked>
    <label for="__yes__""> yes </label>
    <input type="__radio / checkbox__" name="yes-no" value="no">
    <label for="__no__""> no </label>

    When a form is submitted, the data is then sent to the server and includes for (for) the options selected. input types radio and checkbox report their values from the value attribute. The data recieved would be yes-no = yes

    If the value attribute is ommitted, the submitted form uses the default value. Which is "on". That may not be ideal for your purposes. yes-no = yes is more intuitive that yes-no = on. Override this with the checked attribute. The required attribute makes input manditory.

    Opacity

    There are many ways to make things translucent.

    The opacity CSS property sets the opacity of an element. Opacity is the degree to which content behind an element is hidden, and is the opposite of transparency.

    opacity: 0.9 opacity: 90%
    /* Global values */
    opacity: inherit;
    opacity: initial;
    opacity: revert;
    opacity: unset;

    Alpha-Values

    A number in the range 0.0 to 1.0, inclusive, or a percentage in the range 0% to 100%, inclusive, representing the opacity of the channel (that is, the value of its alpha channel). Any value outside the interval, though valid, is clamped to the nearest limit in the range.

    RGBA background: rgba(0, 0, 0, 0.4);

    HEX codes #rrggbbaa; or #rgba

    HSL (sometimes hsla(0, 0%, 0%, 0.4)) functional notation expresses a given color according to its hue, saturation, and lightness components. An optional alpha component represents the color's transparency.

    The Box Model

    Everything in CSS has a box around it, and understanding these boxes is key to being able to create layouts with CSS, or to align items with other items.

    Block and inline boxes refer to how the box behaves in terms of page flow and in relation to other boxes on the page. Boxes also have an inner display type and an outer display type.

      If a box has an outer display type of Block, it will behave in the following ways:

    • The box will break onto a new line.
    • The box will extend in the inline direction to fill the space available in its container. In most cases this means that the box will become as wide as its container, filling up 100% of the space available.
    • The width and height properties are respected.
    • Padding, margin and border will cause other elements to be pushed away from the box

    Some HTML elements, such as <h1> and <p>, use block as their outer display type by default.

      If a box has an outer display type of inline:

    • The box will not break onto a new line.
    • The width and height properties will not apply.
    • Vertical padding, margins, and borders will apply but will not cause other inline boxes to move away from the box.
    • Horizontal padding, margins, and borders will apply and will cause other inline boxes to move away from the box.

    Some HTML elements, such as <a>, <span>, <em>, and <strong> use inline as their outer display type by default.

    The type of box applied to an element is defined by display property values such as block and inline, and relates to the outer value of display.
